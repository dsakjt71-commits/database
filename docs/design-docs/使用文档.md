# Tyouqu.Database 使用文档

## 基本用法

业务 Repository 通过依赖注入获取 `ISqlTemplateExecutor` 或 `IDbExecutor`。

推荐优先使用 SQL 模板编号：

```csharp
public class UserRepository
{
    private readonly ISqlTemplateExecutor _db;

    public UserRepository(ISqlTemplateExecutor db)
    {
        _db = db;
    }

    public Task<User?> GetByUsernameAsync(string username)
    {
        return _db.QuerySingleOrDefaultByIdAsync<User>(
            "auth.user.getByUsername",
            new { Username = username });
    }
}
```

对应 SQL 文件：

```text
sql/sqlserver/auth/auth.user.getByUsername.sql
sql/mysql/auth/auth.user.getByUsername.sql
sql/postgresql/auth/auth.user.getByUsername.sql
sql/sqlite/auth/auth.user.getByUsername.sql
```

SQL 文件内容：

```sql
select *
from users
where username = @Username;
```

SQL ID 等于文件名去掉 `.sql`，文件内容中不要写 `-- @id xxx`。

## 执行查询

查询列表：

```csharp
var users = await _db.QueryByIdAsync<User>(
    "auth.user.listActive",
    new { Status = 1 });
```

查询单条：

```csharp
var user = await _db.QuerySingleOrDefaultByIdAsync<User>(
    "auth.user.getById",
    new { UserId = userId });
```

执行命令：

```csharp
var affectedRows = await _db.ExecuteByIdAsync(
    "auth.user.updateLastLogin",
    new
    {
        UserId = userId,
        LastLoginAt = DateTimeOffset.UtcNow
    });
```

## 参数规范

SQL 模板中统一使用 `@ParameterName`：

```sql
select *
from users
where email = @Email;
```

C# 参数名保持一致：

```csharp
await _db.QuerySingleOrDefaultByIdAsync<User>(
    "auth.user.getByEmail",
    new { Email = email });
```

命名建议：

- 参数名使用 PascalCase。
- 数据库字段名使用当前项目约定，可以是 `snake_case` 或 `PascalCase`。
- SQL ID 使用小写点分格式，例如 `auth.user.getByEmail`。

## 分页查询

分页请求模型：

```csharp
public sealed record PageRequest(int PageIndex, int PageSize);
```

分页返回模型：

```csharp
public sealed class PagedResult<T>
{
    public IReadOnlyList<T> Items { get; init; } = Array.Empty<T>();

    public long TotalCount { get; init; }

    public int PageIndex { get; init; }

    public int PageSize { get; init; }
}
```

使用示例：

```csharp
var result = await _db.QueryPagedByIdAsync<User>(
    "auth.user.search",
    new { Keyword = keyword, Status = status },
    new PageRequest(pageIndex, pageSize));
```

SQL 模板：

```sql
select *
from users
where (@Keyword is null or username like @Keyword)
  and (@Status is null or status = @Status)
order by created_at desc;
```

分页 SQL 由方言自动追加：

```text
SQL Server: OFFSET ... FETCH NEXT ...
MySQL: LIMIT ... OFFSET ...
PostgreSQL: LIMIT ... OFFSET ...
SQLite: LIMIT ... OFFSET ...
```

## 事务使用

多个操作需要一致提交时使用事务：

```csharp
await using var tx = await _unitOfWork.BeginTransactionAsync();

await _db.ExecuteByIdAsync("auth.user.create", user);
await _db.ExecuteByIdAsync("auth.audit.write", auditLog);

await tx.CommitAsync();
```

如果中间抛出异常，事务应自动回滚：

```csharp
await using var tx = await _unitOfWork.BeginTransactionAsync();

try
{
    await _db.ExecuteByIdAsync("file.metadata.create", file);
    await _db.ExecuteByIdAsync("file.storage.markUsed", storage);

    await tx.CommitAsync();
}
catch
{
    await tx.RollbackAsync();
    throw;
}
```

## 原始 SQL 执行

组件可以保留原始 SQL 执行能力，用于调试或极少数动态 SQL 场景。

```csharp
var count = await _rawDb.QuerySingleOrDefaultAsync<int>(
    "select count(1) from users where status = @Status",
    new { Status = 1 });
```

生产业务代码推荐使用 SQL ID，不推荐大量直接写 SQL。

## 动态条件建议

第一版建议用 SQL 参数表达简单动态条件：

```sql
select *
from books
where (@Keyword is null or title like @Keyword)
  and (@CategoryId is null or category_id = @CategoryId)
  and (@Status is null or status = @Status)
order by updated_at desc;
```

复杂动态 SQL 后续可以增加模板语法，例如：

```sql
{{#if Keyword}}
and title like @Keyword
{{/if}}
```

但第一版不建议引入复杂模板引擎，避免组件过早复杂化。

## 错误处理

组件应统一抛出数据库组件异常，例如：

```csharp
public class TyouquDatabaseException : Exception
{
    public string? SqlId { get; }

    public string Provider { get; }
}
```

业务层可以捕获并转换为业务错误：

```csharp
try
{
    return await _db.QuerySingleOrDefaultByIdAsync<User>("auth.user.getById", new { UserId = id });
}
catch (TyouquDatabaseException ex)
{
    _logger.LogError(ex, "查询用户失败");
    throw;
}
```
