# Tyouqu.Database 配置文档

## 完整配置示例

```json
{
  "Database": {
    "Provider": "SqlServer",
    "ConnectionString": "Server=192.168.1.2;Database=AuthManagement;User Id=sa;Password=******;TrustServerCertificate=True;",
    "CommandTimeoutSeconds": 30,
    "EnableSensitiveLogging": false,
    "SqlTemplates": {
      "RootPath": "sql",
      "EnableHotReload": false,
      "ReloadMode": "Manual",
      "FailOnDuplicateSqlId": true,
      "FailOnMissingSqlId": true
    },
    "Pool": {
      "MinPoolSize": 0,
      "MaxPoolSize": 100
    }
  }
}
```

## Database.Provider

指定数据库类型。

支持值：

```text
SqlServer
MySql
PostgreSql
Sqlite
```

当前组件已提供这四个 Provider 的连接工厂、SQL 方言和 DI 注册入口。使用 `Tyouqu.Database` 统一入口时，组件会根据该配置自动选择实现。

## Database.ConnectionString

数据库连接字符串。

SQL Server 示例：

```json
{
  "ConnectionString": "Server=192.168.1.2;Database=AuthManagement;User Id=sa;Password=******;TrustServerCertificate=True;"
}
```

MySQL 示例：

```json
{
  "ConnectionString": "Server=192.168.1.2;Port=3306;Database=app_db;User ID=root;Password=******;"
}
```

PostgreSQL 示例：

```json
{
  "ConnectionString": "Host=192.168.1.2;Port=5432;Database=app_db;Username=postgres;Password=******;"
}
```

SQLite 示例：

```json
{
  "ConnectionString": "Data Source=D:\\AppData\\app.db"
}
```

## Database.CommandTimeoutSeconds

SQL 命令超时时间，单位秒。

建议：

```text
普通 Web 服务：30
报表查询：60-120
后台批处理：120-300
```

## Database.EnableSensitiveLogging

是否记录敏感参数。

生产环境必须设置为：

```json
{
  "EnableSensitiveLogging": false
}
```

开启后日志可能出现邮箱、手机号、用户 ID、业务 ID 等参数，仅建议本地调试使用。

## SqlTemplates.RootPath

SQL 模板根目录。

相对路径：

```json
{
  "RootPath": "sql"
}
```

绝对路径：

```json
{
  "RootPath": "D:\\AuthServiceDeploy\\sql"
}
```

推荐生产环境使用部署目录下的固定路径。

目录约定：

```text
sql/
  common/
  sqlserver/
  mysql/
  postgresql/
  sqlite/
```

SQL ID 约定：

```text
auth.app.getById.sql -> auth.app.getById
```

该规则对 SQL Server、MySQL、PostgreSQL、SQLite 都一样。组件只使用文件名作为 SQL ID，不读取文件内的 `-- @id` 标记；文件中出现 `-- @id ` 会被视为错误。

## SqlTemplates.EnableHotReload

是否启用文件变化自动重载。

建议：

```text
本地开发：true
测试环境：true 或 Manual
生产环境：false
```

## SqlTemplates.ReloadMode

支持值：

```text
Startup
Manual
Watch
```

说明：

| 值 | 说明 |
| --- | --- |
| Startup | 仅服务启动时加载 |
| Manual | 启动加载，后续通过管理接口或内部服务手动重载 |
| Watch | 监听 SQL 文件变化并自动重载 |

生产环境推荐：

```json
{
  "ReloadMode": "Manual"
}
```

## SqlTemplates.FailOnDuplicateSqlId

发现重复 SQL ID 时是否启动失败。

推荐设置：

```json
{
  "FailOnDuplicateSqlId": true
}
```

如果 common 和 provider 下有同名 SQL ID，Provider 专用 SQL 覆盖 common SQL，不视为异常。

## SqlTemplates.FailOnMissingSqlId

业务代码请求不存在的 SQL ID 时是否直接抛出异常。

推荐设置：

```json
{
  "FailOnMissingSqlId": true
}
```

## Pool 配置

连接池大多由底层数据库驱动处理。组件可以把配置合并进连接字符串，也可以仅作为文档规范。

SQL Server 示例：

```text
Min Pool Size=0;Max Pool Size=100;
```

MySQL 示例：

```text
MinimumPoolSize=0;MaximumPoolSize=100;
```

PostgreSQL 示例：

```text
Minimum Pool Size=0;Maximum Pool Size=100;
```

## 环境变量覆盖

推荐支持环境变量覆盖：

```text
Database__Provider=SqlServer
Database__ConnectionString=Server=...
Database__SqlTemplates__RootPath=D:\AuthServiceDeploy\sql
Database__SqlTemplates__ReloadMode=Manual
```

Docker Compose 示例：

```yaml
environment:
  Database__Provider: SqlServer
  Database__ConnectionString: Server=192.168.1.2;Database=AuthManagement;User Id=sa;Password=******;TrustServerCertificate=True;
  Database__SqlTemplates__RootPath: /app/sql
  Database__SqlTemplates__ReloadMode: Manual
```
