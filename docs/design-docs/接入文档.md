# Tyouqu.Database 接入文档

## 接入目标

接入 `Tyouqu.Database` 后，业务服务应做到：

- 不重复封装数据库连接。
- 不重复实现事务和分页。
- 不把 SQL 写死在 C# 代码中。
- 可以通过配置切换数据库 Provider。
- 可以通过外置 SQL 文件维护 SQL。

## 安装组件

组件实现后，推荐按需引用 NuGet 包或项目引用。

SQL Server 项目：

```text
Tyouqu.Database.Abstractions
Tyouqu.Database.Dapper
Tyouqu.Database.SqlServer
```

MySQL 项目：

```text
Tyouqu.Database.Abstractions
Tyouqu.Database.Dapper
Tyouqu.Database.MySql
```

PostgreSQL 项目：

```text
Tyouqu.Database.Abstractions
Tyouqu.Database.Dapper
Tyouqu.Database.PostgreSql
```

SQLite 项目：

```text
Tyouqu.Database.Abstractions
Tyouqu.Database.Dapper
Tyouqu.Database.Sqlite
```

如果希望应用只通过配置选择数据库，可以直接引用统一入口：

```text
Tyouqu.Database
```

## Program.cs 注册

当前组件提供两种注册方式。

统一入口，根据 `DatabaseOptions.Provider` 选择 SQL Server、MySQL、PostgreSQL 或 SQLite：

```csharp
using Tyouqu.Database.Abstractions;
using Tyouqu.Database.DependencyInjection;

builder.Services.AddTyouquDatabase(options =>
{
    options.Provider = DatabaseProvider.SqlServer;
    options.ConnectionString = builder.Configuration.GetConnectionString("Default")!;
    options.CommandTimeoutSeconds = 30;
    options.SqlTemplates.RootPath = "sql";
    options.SqlTemplates.EnableHotReload = false;
});
```

也可以只引用某一个 Provider 包并使用专用入口：

```csharp
using Tyouqu.Database.Abstractions;
using Tyouqu.Database.SqlServer.DependencyInjection;

builder.Services.AddTyouquSqlServerDatabase(options =>
{
    options.Provider = DatabaseProvider.SqlServer;
    options.ConnectionString = builder.Configuration.GetConnectionString("Default")!;
    options.SqlTemplates.RootPath = "sql";
    options.SqlTemplates.EnableHotReload = false;
});
```

## SQL 文件放置

项目目录：

```text
YourService/
  sql/
    common/
      audit.create.sql
    sqlserver/
      auth.user.getById.sql
      auth.app.getById.sql
    mysql/
      auth.user.getById.sql
      auth.app.getById.sql
```

部署目录：

```text
D:\YourServiceDeploy\
  app\
  sql\
    common\
    sqlserver\
```

生产环境建议把 SQL 目录放在部署目录下，并纳入备份。

## 在 Repository 中接入

```csharp
public class BookRepository
{
    private readonly ISqlTemplateExecutor _db;

    public BookRepository(ISqlTemplateExecutor db)
    {
        _db = db;
    }

    public Task<Book?> GetByIdAsync(long bookId)
    {
        return _db.QuerySingleOrDefaultByIdAsync<Book>(
            "novel.book.getById",
            new { BookId = bookId });
    }

    public Task<IReadOnlyList<Book>> SearchAsync(string? keyword)
    {
        return _db.QueryByIdAsync<Book>(
            "novel.book.search",
            new { Keyword = string.IsNullOrWhiteSpace(keyword) ? null : $"%{keyword}%" });
    }
}
```

对应 SQL：

文件名：`novel.book.getById.sql`

```sql
select *
from books
where book_id = @BookId;
```

文件名：`novel.book.search.sql`

```sql
select *
from books
where (@Keyword is null or title like @Keyword)
order by updated_at desc;
```

注意：当前组件规定 SQL ID 等于文件名去掉 `.sql`，不支持在文件中用 `-- @id xxx` 声明 SQL ID。文件里出现 `-- @id ` 会直接报错。

## 认证中心迁移接入建议

认证中心当前在 `AppRepository:Mode` 为 `SqlTemplate` 时已经接入 `Tyouqu.Database` 四库统一入口。现有配置默认 `Database:Provider = SqlServer`，所以当前仍连接 SQL Server；后续要切换 MySQL、PostgreSQL 或 SQLite，需要同步准备对应 Provider 目录下的 SQL 模板和连接字符串。

```text
IUserRepository
  -> StoredProcedureUserRepository
  -> SqlTemplateUserRepository
```

通过配置切换：

```json
{
  "AuthCenter": {
    "RepositoryMode": "SqlTemplate"
  }
}
```

注册逻辑：

```csharp
if (repositoryMode == "SqlTemplate")
{
    services.AddScoped<IUserRepository, SqlTemplateUserRepository>();
}
else
{
    services.AddScoped<IUserRepository, StoredProcedureUserRepository>();
}
```

这样迁移时不需要一次性替换所有存储过程。

## 文件中心接入建议

文件中心可以直接定义自己的 SQL ID：

```text
file.metadata.create
file.metadata.getById
file.metadata.search
file.storage.markUsed
file.storage.markDeleted
```

业务代码只依赖 `ISqlTemplateExecutor`，不需要知道底层是 SQL Server 还是 MySQL。

## 小说程序接入建议

小说程序可以定义：

```text
novel.book.create
novel.book.getById
novel.book.search
novel.chapter.create
novel.chapter.listByBookId
novel.reader.bookmark.save
```

SQL 文件：

```text
sql/sqlserver/novel/novel.book.create.sql
sql/sqlserver/novel/novel.book.getById.sql
sql/sqlserver/novel/novel.chapter.listByBookId.sql
sql/mysql/novel/novel.book.create.sql
sql/sqlite/novel/novel.book.create.sql
```

## 接入检查清单

- 已引用正确 Provider 包。
- 已配置 `Database.Provider`。
- 已配置连接字符串。
- SQL 文件目录存在。
- SQL ID 没有重复冲突。
- 启动时 SQL 模板加载成功。
- Repository 不直接写死业务 SQL。
- 生产环境已关闭自动热加载或改为手动重载。
